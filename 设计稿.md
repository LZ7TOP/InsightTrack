# InsightTrack 浏览器插件设计稿

## 一、 项目背景与目标

**InsightTrack** 是一款旨在帮助用户量化和分析网页浏览习惯的浏览器扩展程序。通过精准区分“页面驻留时间”与“实际活跃时间”，它能提供真实的用户注意力分配数据，并通过可视化的方式帮助用户提升时间管理和专注力。

## 二、 核心功能模块

### 1. 数据采集与监控 (核心引擎)

- **活跃度判定机制**：
  - **标签页驻留时间 (Tab Open Time)**：网页被加载后存在于标签栏的总时间。
  - **实际活跃时间 (Active Time)**：用户焦点在该标签页所在窗口，且页面处于前台，同时存在交互行为（如鼠标移动、滚动、点击、键盘输入）的时间。
  - **空闲判定**：当用户无操作超过设定阈值（如3分钟），将自动判定为空闲并暂停活跃时间的计时。
- **域名自动聚合**：自动解析网页 URL，提取主域名或子域名进行归类统计，支持智能识别常见站点的别名。

### 2. 数据可视化仪表盘 (Dashboard)

- **全局概览**：
  - 展示今日、本周、本月的总活跃时间与总驻留时间，以及相比上一周期的变化率。
  - 使用饼图/环形图直观展示访问时间 Top 10 网站的占比。
- **多维度图表展示**：
  - **时间趋势图 (折线图/柱状图)**：展示选定周期内，每天总浏览时间的起伏变化。
  - **时间段分布图 (热力图)**：展示一天 24 小时中的活跃时段分布，帮助用户了解自己的“数字作息规律”。
- **灵活的时间筛选器**：支持快速选择（今天、昨天、过去7天、本月）或通过日历组件自定义任意时间区间。

### 3. 单站点深度分析与比对

- **专属数据报表**：点击具体网站，可下钻查看该站点的专属统计数据（例如：最常访问的子路径、活跃时间峰值时段）。
- **历史数据比对**：
  - 支持用户选取该站点在不同时间段的数据进行对比（例如：本周对比上周）。
  - 比对结果以双柱状图或重叠折线图呈现，帮助用户发现习惯的改变。

### 4. 偏好设置与管理

- **追踪黑名单 (隐私保护)**：
  - 用户可添加不想被记录的网站（支持具体 URL 或通配符如 `*.local`）。黑名单内的网站将完全不纳入时间和活动的统计。
- **灵活的判定阈值**：用户可根据个人习惯，自定义判定“非活跃”的无操作时长。
- **数据管理 (安全与自由)**：
  - 数据全量保留在本地浏览器中，绝不上传，保障隐私。
  - 支持数据的导入、导出（JSON/CSV 格式）以及一键清空历史数据。

## 三、 技术架构与实现建议

- **扩展规范**：采用最新的 Manifest V3 标准。
- **Service Worker (后台脚本)**：负责监听 `chrome.tabs` 和 `chrome.windows` 状态变化，统筹管理时间计算与存储协调。
- **Content Scripts (内容脚本)**：注入网页监听用户真实交互事件（mousemove, scroll, keydown 等），向后台汇报活跃状态。
- **UI 交互层**：
  - **Popup (点击图标的弹窗)**：显示当日快速简报、当前网站统计以及一键加入黑名单的开关。
  - **Options/Dashboard (独立标签页)**：承载详尽的图表库（建议引入 ECharts 或 Chart.js 等数据可视化库）及完整的设置项，可采用 React/Vue 框架构建单页应用。
- **数据存储方案**：考虑到长期记录会积累大量日志数据，推荐使用 `IndexedDB` 存储结构化细粒度数据，`chrome.storage.local` 存储简单的配置项。
